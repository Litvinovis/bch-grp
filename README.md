# БЧ-ГРП — Discord RPG Бот

**БЧ-ГРП** (BCH-GRP) — многопользовательская текстовая RPG-игра, работающая как Discord-бот. Игра написана на Java, использует PostgreSQL в качестве хранилища данных и JDA для взаимодействия с Discord API.

Создана по мотивам легендарной ЧБ-РПГ для чатика безумия.

---

## Команды бота

Все команды начинаются с префикса `+`.

### Основные

| Команда | Описание |
|---|---|
| `+регистрация` | Зарегистрировать персонажа |
| `+помощь` | Полный список команд |
| `+инфо` | Общая информация об игре |
| `+стата` | Информация о вашем персонаже |

### Перемещение и локации

| Команда | Описание |
|---|---|
| `+идти <локация>` | Переместиться в указанную локацию |
| `+локация` / `+локация <название>` | Информация о текущей или указанной локации |
| `+карта` | Карта мира БЧ-ГРП |

### Предметы и инвентарь

| Команда | Описание |
|---|---|
| `+инвентарь` | Список предметов в инвентаре |
| `+инвентарь <предмет>` | Подробная информация о предмете |
| `+предмет <предмет>` | Информация о любом предмете в игре |
| `+купить <предмет>` | Купить предмет в магазине |
| `+использовать <предмет>` | Использовать предмет из инвентаря |
| `+продать <предмет>` | Продать предмет из инвентаря |

### Таверна (мини-игры)

| Команда | Описание |
|---|---|
| `+кости <ставка>` | Кубики (ставка 1–100) |
| `+рулетка <ставка> <красный/черный/0-36>` | Рулетка (x2 или x35) |
| `+кнб <ставка> <камень/ножницы/бумага>` | Камень-ножницы-бумага |
| `+число <ставка> <1-10>` | Угадай число (x5) |

### Активности

| Команда | Описание |
|---|---|
| `+взять квест` | Получить квест |
| `+выполнить квест [ответ]` | Выполнить квест |
| `+поменять квест` | Сменить квест (стоит 5 монет) |
| `+убить босса` | Атаковать босса текущей локации |
| `+пвп` | PvP с случайным игроком локации (только PvP-зоны) |
| `+бонус` | Ежедневный бонус (стрик 3 дня = +50 монет, 7 дней = редкий предмет) |

### Рейды

| Команда | Описание |
|---|---|
| `+рейд` | Начать рейд на общего босса |
| `+присоединиться` | Вступить в активный рейд |

### Классы и прогресс

| Команда | Описание |
|---|---|
| `+класс <воин/разбойник/маг>` | Выбрать класс с 5 уровня (один раз) |
| `+достижения` | Показать разблокированные достижения |
| `+топ [уровень/деньги/репутация]` | Таблица лидеров top-10 |

### Дуэли

| Команда | Описание |
|---|---|
| `+вызов @игрок` | Вызвать игрока на дуэль |
| `+принять` | Принять вызов на дуэль |
| `+отказать` | Отказаться от дуэли |

### Торговля

| Команда | Описание |
|---|---|
| `+передать @игрок предмет [количество]` | Передать предмет другому игроку |

### Кланы

| Команда | Описание |
|---|---|
| `+новый клан <название>` | Создать клан (требуется 10+ уровень) |
| `+вступить в клан <название>` | Подать заявку в клан (3+ уровень) |
| `+покинуть клан` | Покинуть клан |
| `+принять заявки` | Принять все заявки (для лидера) |
| `+отклонить заявки` | Отклонить все заявки (для лидера) |
| `+клан инфо <название>` | Информация о клане |

### Вспомогательные

| Команда | Описание |
|---|---|
| `+идея <текст>` | Оставить идею / баг-репорт |
| `+статус` | Состояние базы данных |

### Административные (только для admins)

| Команда | Описание |
|---|---|
| `+всеидеи` | Все идеи |
| `+новыеидеи` | Только непросмотренные идеи |
| `+идеяномер <номер>` | Идея по номеру |
| `+идеястатус <номер> <статус>` | Изменить статус идеи |

## Стек технологий

- **Java 25**
- **JDA 5.6.1** — Discord API
- **PostgreSQL 16** — хранилище данных (таблицы: players, locations, items, ideas, clans, bosses)
- **HikariCP 6.3.0** — пул соединений
- **Logback** — логирование
- **SnakeYAML** — парсинг конфигурации
- **Maven** + **maven-assembly-plugin** — сборка fat-jar

## Конфигурация

Конфигурация загружается из `application.yml` (не хранится в репозитории — содержит чувствительные данные).

Создайте файл `src/main/resources/application.yml` со следующей структурой:

```yaml
discord:
  allowed-channel-ids:   # ID каналов, где принимаются команды
    - "CHANNEL_ID"
  admin-ids:             # ID пользователей с правами администратора
    - "ADMIN_USER_ID"

db:
  url: "jdbc:postgresql://127.0.0.1:5432/bchgrp"
  username: "bchgrp"
  password: "ваш_пароль"
```

**Токен Discord** передаётся через переменную окружения (не в конфиге):

```bash
export BCHGRP_DISCORD_TOKEN="ваш_токен"
```

## Локальный запуск

### Требования

- JDK 25+
- Maven 3.8+
- PostgreSQL 16+

### Сборка

```bash
mvn package -DskipTests
```

### Запуск

```bash
export BCHGRP_DISCORD_TOKEN="ваш_токен_discord"
java -jar target/bchgrp-1.0-SNAPSHOT-jar-with-dependencies.jar
```

Схема БД создаётся автоматически при первом запуске (`schema.sql`).

## Деплой через GitHub Actions

Репозиторий использует GitHub Actions с self-hosted runner для автоматического деплоя на сервер.

### Схема CI/CD

1. **Push в ветку `master`** → запускается workflow `.github/workflows/deploy.yml`
2. **Build & Test** — `mvn package -B` (сборка + тесты)
3. **Deploy** — собранный fat-jar копируется в `/opt/BCHGRP/bchgrp.jar`
4. **Рестарт сервиса** — `systemctl restart bot-bchgrp`
5. **Smoke test** — проверка что сервис поднялся; при падении — автоматический откат на предыдущий jar
6. **Уведомление в Telegram** — результат деплоя

### Необходимые GitHub Secrets

| Secret | Описание |
|---|---|
| `TELEGRAM_BOT_TOKEN` | Токен Telegram-бота для уведомлений |
| `TELEGRAM_CHAT_ID` | ID чата для уведомлений |

**Токен Discord** хранится в `.env`-файле на сервере и загружается systemd-сервисом (`EnvironmentFile`), а не в GitHub Secrets.

### Структура деплоя на сервере

```
/opt/BCHGRP/
├── bchgrp.jar                    # Текущий запущенный jar
├── bchgrp.jar.backup.YYYYMMDD_HHMM  # Бэкапы (автоматически)
└── application.yml               # Runtime-конфиг (не в репозитории)
```

Сервис: `systemctl status bot-bchgrp`
